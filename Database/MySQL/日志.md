# redo log
记录 **物理修改**
InnoDB 独有, 循环写
*“先写日志，再写数据”* 在空闲时将修改写入磁盘
- 提高数据库的事务处理性能 (避免每次提交都刷数据页)。
- 保证事务的持久性 (即使系统崩溃也能恢复已提交的事务)。
- 实现崩溃恢复 (通过 Redo Log 将数据库恢复到崩溃前的状态)。
